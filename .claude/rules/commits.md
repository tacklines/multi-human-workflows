# Commit Conventions

This is a new project with no commit history yet. Use conventional commits going forward.

## Format

```
<type>: <short description>
```

Types: feat, fix, refactor, test, docs, chore, style

## Examples

- `feat: add drag-and-drop YAML file loading`
- `fix: correct schema validation for optional fields`
- `test: add comparison edge case tests`
- `refactor: extract event grouping into shared utility`

## Rules

- Subject line under 72 characters
- Imperative mood ("add" not "added")
- No period at the end of the subject line
- Body optional; use when the "why" is not obvious from the diff
